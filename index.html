Course Vision

Structure first.
Creativity follows.

TechCommons is a 10-week introductory Python course for children aged 9 to 12 with no prior coding experience. Sessions are structured, beginner-friendly, and designed to culminate in every student building their own short playable game from scratch.

🎮

A real final project

Students learn loops, conditions, variables, and functions, then combine them into a game that feels personally theirs.

🧭

Predictable weekly rhythm

Each two-hour session has a focused content phase followed by a hands-on activity phase with room for guided or self-directed projects.

🐞

Bugs are part of the culture

Errors are treated as evidence that students are coding for real. Facilitators model debugging calmly and openly.

🌱

Accessible by design

Each student pays $10 per lesson to cover the meeting room, a remarkably low fee designed to keep the course open to students across socioeconomic backgrounds.

" If your code has a bug, congratulations - you're a programmer. The skill is learning how to read an error, make a guess, and try again.
— TechCommons Founder

week_10_game.py Week 10 — Game 
# Week 10 — Text adventure game
# Built using everything from weeks 1–9

import random

def start_game():
    print("\n=== THE LOST KEY ===")
    print("Your mission: find the key before sunset.")
    rooms = {
        "kitchen": "A counter, a drawer, and the fridge.",
        "lounge":  "Sofa cushions and a coffee table.",
        "garden":  "A flowerpot and a garden chair.",
    }
    key_room = random.choice(list(rooms.keys()))
    found    = False

    while not found:
        choice = input("Search (kitchen/lounge/garden/quit): ").lower()
        if choice == "quit":
            print("Game over."); break
        elif choice in rooms:
            print(f"You check the {choice}. {rooms[choice]}")
            if choice == key_room:
                print("You found the key! You win.")
                found = True
        else:
            print("That's not a valid room.")

Final Project

A game worth
showing someone.

The final game is designed in advance by the teaching team, but students make it their own through theme, story, dialogue, names, and optional features.

The goal is not technical flashiness. The goal is ownership: each student should leave with a playable Python project that feels like theirs, built from the concepts they practised across the course.

🎲

Shared structure

The core logic is common to the class: choices, scoring, loops, conditions, and win or lose outcomes.

🎨

Personal aesthetic

Students choose the world, characters, names, dialogue, and tone of their own game.

💻

Playable by Week 10

The final weeks focus on getting every project to a complete, runnable state in Trinket.io.

🧪

Debugging support

Facilitators help students read errors, test guesses, and fix problems without taking over the keyboard.

👏

Showcase, not assessment

Week 10 is a celebration where students play each other's games and experience a real "look what I made" moment.
